## What's New

### Artifact Timestamps (ISS-026)

Pipeline artifacts now include a visible ISO 8601 timestamp, enabling operators to immediately identify when each artifact was produced and trace the sequence of review cycles.

#### What Changed

**Before:** Feature artifacts (prd.md, architecture.md, security-audit.md, review files) had no metadata indicating when they were generated. Operators could not distinguish between an artifact created in phase 2 versus one regenerated in phase 6, making it difficult to understand which artifacts were stale or how review cycles related in sequence.

**After:** All pipeline-generated artifacts include a `**Generated:** <ISO 8601>` line immediately after the document's top-level heading. On regeneration, the timestamp is automatically updated to the current time rather than preserved from the earlier run, ensuring timestamps always reflect when the artifact was actually produced.

#### Placement Example

```markdown
# Feature: New Payment API
**Generated:** 2026-04-13T14:30:00Z

## Overview
```

#### Scope

The convention applies to all feature artifact types:
- `prd.md` (Phase 1 Specify)
- `architecture.md` (Phase 2 Architect)
- `security-audit.md` (Phase 4 Security Gate)
- `review.md` and `review-claude-*.md` (Phase 6 Review)
- `review-codex-code-*.md`, `review-codex-architecture-*.md`, `review-codex-tests-*.md`, `review-codex-prd-*.md` (Codex reviews)

#### Documentation

The convention is documented in `docs/CLAUDE.md` under Code Conventions as the canonical requirement. All command templates and skill files include cross-references to ensure consistency across the framework.
