## What's New

### Invariants-Audit Skill for Cross-Layer Semantic Review (ISS-001)

Reviewers now have a reusable skill that teaches them to detect cross-layer semantic mismatches — contradictions between spec, implementation, hooks, and tests that survive passing test suites and normal code review. This addresses a recurring defect class that has caused 5+ production incidents.

The skill provides:
- A **5-step invariant review method** that systematically checks for state-machine bugs, blocked/rejected paths, spec-vs-implementation contradictions, fixture-template mismatches, and test-behavior gaps
- **5 review categories** covering state-machine transitions, failure paths, cross-artifact contradictions, fixture mismatches, and syntax-only tests
- **Clear trigger conditions** distinguishing when to use invariants-audit versus normal code review, architecture review, or QA review
- **Integration into 4 Claude reviewer commands** (code-review, architect, security-gate, test-design) and **4 Codex reviewer prompts** (review-code, review-prd, review-architecture, review-test-design)

#### Progressive Disclosure Pattern

The skill demonstrates the new progressive-disclosure pattern: the main SKILL.md file stays compact (64 lines), while detailed content lives in a sibling reference file (`review-categories.md`). This keeps core instructions crisp while enabling depth for reviewers who need it.
