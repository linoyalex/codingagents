## What's New

### QA Test Quality Hardening (ISS-043, ISS-045, ISS-049)

Phase 3 test design now produces higher-quality test shells that catch real defects on first pass, reducing ~50% rework from Codex review feedback. Three recurring quality gaps are addressed:

#### Symmetric Testing (ISS-043)
- QA agents are instructed to test ALL enumerated components when the architecture lists multiple items, not just one representative
- TDD skill's "What to Test First" list includes a `[symmetric-coverage]` entry ensuring symmetric requirements are checked across all consumers

#### Adversarial Contract Testing (ISS-045)
- QA agents now test that safety contracts cannot be trivially evaded — commented-out code, escape hatches, string matches on dead code
- TDD skill includes a `[contract-robustness]` entry for adversarial contract checks

#### Fixture-Driven Test Strategy (ISS-049)
- New 3-way artifact-type routing: declarative artifacts → structural checks, executable code → fixture-driven behavioral tests, config → schema validation
- Hybrid artifacts (executable + declarative) route by precedence: behavioral tests first
- All guidance is stack-agnostic with `node --test` + `pytest` examples

#### Progressive Disclosure
- New sibling reference file `skills/tdd/test-quality-rules.md` holds expanded guidance, keeping `SKILL.md` under the 120-line budget
- Follows the pattern established by the code-review skill (ISS-039)
