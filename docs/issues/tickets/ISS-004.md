# ISS-004: Phase 4 verification logic does not match the canonical security-audit template

- **Type:** Bug
- **Priority:** P2 — Medium
- **Label:** `pipeline`, `testing`
- **Created:** 2026-04-06
- **Reporter:** Codex
- **Feature area:** Verification gate skill, Phase 4 security audit template

## Description

The verification-gate skill's Phase 4 regex expected one heading format for BLOCKING findings, but the canonical security-audit template used a different format. This caused verification to pass even when BLOCKING findings were present.

## Acceptance Criteria

- [ ] `skills/verification-gate/SKILL.md` Phase 4 regex matches both heading format (`#### BLOCKING:`) and list format (`- BLOCKING:`)
- [ ] Test fixture updated to canonical heading format
- [ ] Regression tests cover both pass and fail cases

## Notes

Resolved 2026-04-06. Regex updated in verification-gate skill. Regression tests in `tests/node/verification-gate.test.js` cover both pass and fail cases.
