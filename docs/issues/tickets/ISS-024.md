# ISS-024: Add reviewer independence and boundary-tracing rules to code-review skill

- **Type:** Feature
- **Priority:** P1 — High
- **Label:** `framework`, `skills`, `review-quality`, `pipeline-quality`
- **Created:** 2026-04-11
- **Reporter:** Linoy (via global memory audit of cross-project learnings)
- **Feature area:** `skills/code-review/SKILL.md`, `commands/review.md`

## Description

The code-review skill says "read the PR description and linked spec first" but does not warn against anchoring on the developer's framing. This caused a reviewer to APPROVE a branch that Codex immediately flagged with BLOCKING findings. The reviewer verified each item on the developer's handoff constraint list and stopped, never asking the adjacent question.

Two related sub-problems:

1. **Reviewer anchoring on developer handoff.** The reviewer confirmed the developer's checklist items rather than re-deriving coverage from the PRD. The bug lived in the symbol the developer didn't touch — adjacent to the one they claimed was fixed.

2. **Boundary transformation tracing missing.** Schema validation silently stripped a field. The review checked that a function was called but not that data survived the transformation chain. A 2-second check of the schema would have caught it.

## Acceptance Criteria

- [x] **AC1 (Reviewer independence section):** `skills/code-review/SKILL.md` adds a "Reviewer Independence" section with these rules:
  - Read PRD acceptance criteria before reading the developer handoff
  - Treat developer claims ("fixed in X", "covered by test Y") as hypotheses to falsify
  - For every "fixed in <symbol>" claim, grep for adjacent symbols that operate on the same data
  - Never assert "test X covers path Y" without opening the test and reading the fixture
  - Cross-read PRD wording against implementation phrases verbatim — flag drift
- [x] **AC2 (Boundary tracing rule):** The skill adds: "For any new field flowing through a parse/validate/transform chain, verify the schema includes it. Trace one test case mentally: what goes in → what the schema keeps → what the consumer sees."
- [x] **AC3 (Review command update):** `commands/review.md` adds: "Read docs/features/$ARGUMENTS/prd.md before .claude/handoff.json to build independent coverage expectations."
- [x] **AC4 (No regression):** Existing tests continue to pass. Skill stays within size budget.

## Notes

### Evidence

- Reviewer APPROVED a branch; Codex found one BLOCKING and one MAJOR finding. Root cause: reviewer verified the developer's handoff constraint list and stopped (closet-designer preset-storage-parity, 2026-04-07).

- Zod's default strip mode silently dropped a property not in the schema. Review checked the function was called but not that data arriving at it still contained the needed field (closet-designer consistent-design-labels).
