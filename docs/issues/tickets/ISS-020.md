# ISS-020: Prefix generated PRD filenames with the feature slug

- **Type:** Architecture
- **Priority:** P3 — Low
- **Label:** `framework`, `artifacts`, `naming`, `pipeline`
- **Created:** 2026-04-07
- **Reporter:** Linoy
- **Feature area:** Feature artifacts, command conventions, pipeline file discovery

## Description

The current pipeline writes feature PRDs to `docs/features/<feature>/prd.md`. That keeps
directories tidy, but it also makes PRD files visually generic in editor tabs, search
results, review tools, and copied paths. A feature-prefixed filename would make the PRD
artifact easier to identify at a glance and align better with the already-prefixed review
artifacts such as `review-codex-architecture-<feature>.md`.

We should add support for generating PRD files with the feature slug prefixed in the
filename, for example `docs/features/<feature>/<feature>-prd.md`, and update the pipeline
conventions and command references accordingly.

## Acceptance Criteria

- [ ] Define the canonical PRD filename convention for feature folders using a
      feature-prefixed name
- [ ] Update the relevant commands, skills, and docs that currently assume
      `docs/features/<feature>/prd.md`
- [ ] Update any status, review, security, or verification flows that discover PRD files
      by fixed path
- [ ] Preserve a deterministic way for agents and scripts to locate the active feature PRD
      without guesswork
- [ ] Decide whether backward compatibility is required for existing `prd.md` artifacts and
      document that migration strategy
- [ ] Add or update tests for any file-discovery or command-contract behavior affected by
      the naming change

## Notes

This is primarily a naming and artifact-discovery convention change. The main risk is not
the rename itself, but breaking all the places in the pipeline that currently hardcode
`prd.md`.

Likely areas to review:
- `commands/specify.md`
- `commands/architect.md`
- `commands/test-design.md`
- `commands/security-gate.md`
- `commands/implement.md`
- `commands/review.md`
- `commands/document.md`
- `commands/status.md`
- skills and tests that reference `docs/features/<feature>/prd.md`
