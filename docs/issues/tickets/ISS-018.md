# ISS-018: Standardize skill naming on gerund form

- **Type:** Architecture
- **Priority:** P3 ‚Äî Low
- **Label:** `framework`, `skills`, `documentation`
- **Created:** 2026-04-07
- **Reporter:** Linoy (via Claude research on ISS-010)
- **Feature area:** `skills/`, naming conventions

## Description

Anthropic's official skill authoring guidance recommends using **gerund form** (verb + -ing) for skill names, because it clearly describes the activity or capability the skill provides. Our current skill names are inconsistent: `prd-writing` is gerund-form, but `architecture-decision`, `tdd`, `code-review`, `security-audit`, `verification-gate`, `release-docs`, `structured-logging`, `backlog-management`, and `claude-api` are noun-form.

Either standardize on gerund form to match Anthropic guidance, or document an explicit decision to keep noun form (with justification) so future maintainers don't drift back and forth.

## Acceptance Criteria

- [ ] Decision recorded: standardize on gerund form OR document why we keep noun form
- [ ] If standardizing: every skill in `skills/` and `.claude/skills/` is renamed (directory, frontmatter `name`, all references in commands and tests)
- [ ] If keeping noun form: a one-line note in `docs/CLAUDE.md` Naming section explains the divergence from Anthropic guidance
- [ ] Any contract test asserting on skill names is updated
- [ ] No regression in `node --test tests/node/` or `bash tests/test-command-contracts.sh`

## Notes

Suggested gerund-form mapping if we standardize:
- `prd-writing` ‚Üí unchanged
- `architecture-decision` ‚Üí `deciding-architecture` or `writing-architecture-decisions`
- `tdd` ‚Üí `practicing-tdd` (or keep as acronym ‚Äî TDD is well-known)
- `code-review` ‚Üí `reviewing-code`
- `security-audit` ‚Üí `auditing-security`
- `verification-gate` ‚Üí `verifying-phase-output`
- `release-docs` ‚Üí `writing-release-docs`
- `structured-logging` ‚Üí `logging-structured` (awkward ‚Äî argues for keeping noun form)
- `backlog-management` ‚Üí `managing-backlog`

The awkwardness of some gerund renames (`logging-structured`) is a real argument for keeping noun form. This is a P3 because the cost of inconsistency is low ‚Äî names are stable references, not under context pressure.
