# ISS-016: Document intentional divergence from Anthropic slash command guidance

- **Type:** Architecture
- **Priority:** P3 — Low
- **Label:** `framework`, `documentation`, `pipeline`
- **Created:** 2026-04-07
- **Reporter:** Linoy (via Claude research on ISS-010)
- **Feature area:** `commands/`, framework design philosophy

## Description

Anthropic's published guidance on Claude Code slash commands explicitly states: *"Use slash commands as simple, personal shortcuts, not as a replacement for building a more intuitive CLAUDE.md and better-tooled agent."* (Builder.io blog, Anthropic webinar). The standard pattern is short, single-purpose commands like `/test`, `/format`, `/lint`.

Our framework uses slash commands very differently: `/specify`, `/architect`, `/test-design`, `/security-gate`, `/implement`, `/review`, `/document` are full phase orchestrators that:
- spawn dedicated subagents
- enforce session/model selection
- write `.claude/handoff.json` for the next phase
- run verification gates
- print the next pipeline step

This is novel and arguably better for our use case (a phase-gated pipeline needs orchestration that simple shortcuts cannot provide), but it is a deliberate divergence from official guidance. Without documentation, a future maintainer who reads Anthropic's docs may try to "simplify" our commands back into shortcuts and break the pipeline contract.

  - Why our use case justifies the divergence (phase-gated pipeline, not ad-hoc tool use)

  - The fallback if the divergence proves wrong (move orchestration into hooks or a dedicated runner)

## Notes

This is a P3 because nothing is broken — but the absence of a documented decision is a maintenance landmine. The first time someone runs `claude --help` and reads Anthropic's "keep slash commands simple" guidance, they will be tempted to simplify ours, and there is currently no record of why they shouldn't.

Related: ISS-001 (invariants-audit) would catch the regression if a "simplification" PR landed and broke the phase contract — but documentation is cheaper than detection.
