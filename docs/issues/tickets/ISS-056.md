# ISS-056: Auto-migrate handoff.json on upgrade when required fields are missing

- **Type:** Feature
- **Priority:** P3 — Low
- **Label:** `framework`, `deployment`, `upgrade`
- **Created:** 2026-04-15
- **Reporter:** Linoy
- **Feature area:** `upgrade.sh`, `schemas/handoff.schema.json`

## Description

When a project upgrades codingagents mid-feature-cycle, the existing `.claude/handoff.json` may be missing fields that became required in newer versions (e.g., `source_spec` was added in v5.5). The upgrade succeeds, but the next pipeline command fails because the handoff doesn't pass validation.

ISS-007 adds a warning for this case. This ticket adds automatic migration: `upgrade.sh` detects missing required fields and infers them from existing handoff data.

## Acceptance Criteria

- [ ] **AC1:** When `upgrade.sh` detects an existing `.claude/handoff.json` with missing required fields, it attempts to infer and add them (e.g., `source_spec` from `relevant_files` PRD path).
- [ ] **AC2:** Inferred fields are logged with a warning: "Migrated handoff.json: added source_spec (inferred from relevant_files)."
- [ ] **AC3:** If a field cannot be inferred, the warning says so and recommends manual fix.
- [ ] **AC4:** A `--no-migrate` flag skips handoff migration.
- [ ] **AC5:** Existing tests pass.

## Notes

- Low priority — ISS-007's warning is sufficient for now.
- This should use the handoff schema (`schemas/handoff.schema.json`) to detect missing required fields rather than hardcoding field names.
- Discovered when upgrading Closet designer from v5.5 to v5.9: Phase 5 handoff was missing `source_spec`, causing `/review` to halt.
