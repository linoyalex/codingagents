# ISS-025: Add adversarial self-review checkpoint to Phase 5 verification

- **Type:** Feature
- **Priority:** P2 — Medium
- **Label:** `framework`, `skills`, `testing`, `verification`
- **Created:** 2026-04-11
- **Reporter:** Linoy (via global memory audit of cross-project learnings)
- **Feature area:** `skills/verification-gate/SKILL.md`, `skills/tdd/SKILL.md`

## Description

After implementing a feature, the developer currently runs verification checks (tests pass, lint clean, handoff written) but does not systematically challenge their own work. Tests that pass on the first try can create false confidence — they prove fixture-implementation consistency, not implementation-production consistency.

This ticket adds an adversarial self-review step to the Phase 5 verification gate and the TDD GREEN step.

## Acceptance Criteria

- [ ] **AC1 (TDD GREEN checkpoint):** `skills/tdd/SKILL.md` adds to the GREEN step: "Before committing GREEN, walk each AC and ask: if a malicious tester wanted to fail this AC while keeping my tests green, what would they do? If an answer exists, that test is missing."
- [ ] **AC2 (Fixture validation in Phase 5):** `skills/verification-gate/SKILL.md` adds to Phase 5 verification: "Validate that test fixtures use production-correct types/values, not stand-in shapes."
- [ ] **AC3 (First-try suspicion):** The TDD skill adds: "Be slightly suspicious when tests pass on the first try. Tests passing means fixture-implementation consistency, not implementation-production consistency. If a critical AC's test went green immediately, double-check that the test exercises the path the AC describes."
- [ ] **AC4 (No regression):** Existing tests continue to pass. Skills stay within size budget.

## Notes

### Evidence

- Tests passed on first try because fixtures used wrong data types — internally consistent but not production-correct. 15/15 tests passed but Codex found a BLOCKING bug (closet-designer preset-storage-parity).
- Self-review in the same session as implementation produced false confidence — the developer was invested in their own code being right (closet-designer consistent-design-labels).
