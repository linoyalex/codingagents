# ISS-041: Existing checkpoint.test.js fixtures must include source_spec after schema change

- **Type:** Bug
- **Priority:** P1 — High
- **Label:** `pipeline`, `testing`
- **Created:** 2026-04-13
- **Reporter:** code-reviewer agent (RCA from review-hardening Phase 6)
- **Feature area:** `tests/fixtures/handoff/`, `tests/node/checkpoint.test.js`

## Description

The review-hardening branch added `source_spec` as a required field in `schemas/handoff.schema.json`, but did not update the existing test fixtures in `tests/fixtures/handoff/`. Codex reproduced this: `node --test tests/node/checkpoint.test.js` fails with 2 test failures because `valid.json` and `unexpected-property.json` lack `source_spec`.

This is a direct violation of the project constraint: "Never remove or skip an existing test to make the suite pass." The schema change effectively broke existing tests without updating them.

## Acceptance Criteria

- [ ] **AC1:** `tests/fixtures/handoff/valid.json` includes a valid `source_spec` field.
- [ ] **AC2:** `tests/fixtures/handoff/unexpected-property.json` includes a valid `source_spec` field.
- [ ] **AC3:** `node --test tests/node/checkpoint.test.js` passes with zero failures.
- [ ] **AC4:** Any other fixture files that reference the handoff schema also include `source_spec`.
