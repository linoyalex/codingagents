# ISS-055: Rename hook scripts to .cjs for ESM project compatibility

- **Type:** Bug
- **Priority:** P1 — High
- **Label:** `framework`, `deployment`, `compatibility`
- **Created:** 2026-04-15
- **Reporter:** Linoy
- **Feature area:** `hooks/`, `init.sh`, `upgrade.sh`, `settings.json`

## Description

All four hook scripts (`checkpoint.js`, `resolve-feature.js`, `restore-context.js`, `archive-context.js`) use `require()` (CommonJS). When installed into a target project with `"type": "module"` in `package.json`, Node.js treats `.js` files as ESM and the hooks fail with an import error.

Discovered when upgrading the Closet designer project (which uses `"type": "module"`) from v5.5 to v5.9. `resolve-feature.js` failed immediately on invocation.

## Fix

Rename all hooks from `.js` to `.cjs`. Node.js always treats `.cjs` as CommonJS regardless of the project's `package.json` `type` field. No code changes needed — only file renames and reference updates.

No ESM rewrite — these are simple scripts using `fs` and `path`. ESM features (top-level await, tree-shaking) add no value for CLI hook scripts.
