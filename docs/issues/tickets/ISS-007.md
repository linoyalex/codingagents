# ISS-007: Force upgrade and clean reinstall with backup support

- **Type:** Feature
- **Priority:** P2 — Medium
- **Label:** `deployment`, `framework`
- **Created:** 2026-04-06
- **Reporter:** Linoy
- **Feature area:** init.sh, upgrade.sh, install lifecycle

## Description

Add two capabilities to the install/upgrade scripts:

1. **`upgrade.sh --force`** — Force an upgrade by overwriting an existing setup, even if the target version matches the installed version. Currently upgrade.sh skips files that already exist or match the current version, which makes it impossible to fix a corrupted install without manual intervention.

2. **`init.sh --clean`** — Clean reinstall mode that removes an existing `.claude/` pipeline setup and performs a fresh installation. This is useful when the installed shape has drifted from the source (e.g. manual edits to installed agents, skills, or commands).

In both cases, any existing files that would be overwritten must be backed up before replacement:
- Back up to `.claude/backup/<timestamp>/` preserving the original directory structure
- Back up agents, skills, commands, helper scripts, schemas, and settings.json
- Print a summary of what was backed up and where

## Acceptance Criteria

- [ ] `upgrade.sh --force` overwrites all installed pipeline files regardless of version match
- [ ] `init.sh --clean` removes existing `.claude/agents/`, `.claude/commands/`, `.claude/skills/`, `.claude/helpers/`, `.claude/schemas/`, and `.claude/settings.json` before reinstalling
- [ ] Before any overwrite or removal, existing files are copied to `.claude/backup/<ISO-timestamp>/` preserving directory structure
- [ ] A summary is printed listing backed-up files and the backup location
- [ ] `init.sh --clean` on a repo with no existing install behaves identically to `init.sh` (no error, no empty backup)
- [ ] `upgrade.sh --force` without an existing install exits with an error ("nothing to upgrade")
- [ ] Backup directory is added to `.gitignore-template` as a runtime artifact
- [ ] Existing tests in `test-install-scripts.sh` continue to pass
- [ ] New test cases cover: force upgrade, clean reinstall, backup creation, no-op on missing install

## Notes

- `settings.local.json` (user permissions) must NOT be removed or overwritten during clean reinstall — only `settings.json` (hooks config) should be replaced.
- Consider whether `.claude/agent-memory/` should be preserved during clean reinstall (it contains persistent agent knowledge, not installed framework files).
