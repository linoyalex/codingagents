# ISS-031: Extend `/document` to update README and other project documentation artifacts

- **Type:** Feature
- **Priority:** P2 — Medium
- **Label:** `framework`, `commands`, `documentation`, `pipeline`
- **Created:** 2026-04-11
- **Reporter:** Linoy
- **Feature area:** `commands/document.md`, `skills/release-docs/SKILL.md`

## Description

The `/document` command (Phase 7) currently updates three artifacts after a feature merge:

1. `CHANGELOG.md` — one bullet per AC
2. `release-notes/` — a new release note file
3. `CLAUDE.md` (or `docs/CLAUDE.md`) — conventions and timestamp

It does **not** update the project `README.md` or any other user-facing documentation. For a framework like codingagents, this means the README drifts from reality after every feature cycle — new commands, changed behavior, added skills, or revised workflow steps are not reflected until someone manually updates it.

This ticket extends `/document` to optionally update `README.md` and other configurable documentation artifacts as part of Phase 7, so that user-facing docs stay current without a separate manual pass.

## Acceptance Criteria

- [ ] **AC1 (README update):** `/document` checks if the feature introduced or changed user-facing behavior (new commands, changed workflow, added skills, modified install steps). If so, it updates the relevant sections of `README.md`.
- [ ] **AC2 (Scoped reads):** The documentation-specialist reads only the feature's `prd.md`, existing `README.md`, and the diff summary — not `src/`. The update is derived from the spec, not the implementation.
- [ ] **AC3 (Additive only):** README updates are additive or corrective — the agent does not rewrite sections unrelated to the feature. Changes are limited to sections that reference concepts touched by the feature.
- [ ] **AC4 (Skill-level artifact registry):** `skills/release-docs/SKILL.md` includes an `## Artifact Registry` section with a table listing documentation artifacts to update. Each entry specifies: artifact name, file path, and a pointer to a sibling reference file containing the update rules. Default entries: `README.md`, `RELEASE.md`. The registry is loaded on demand when `/document` runs — not always-loaded in CLAUDE.md.
- [ ] **AC4a (Sibling reference files for each artifact):** Each registered artifact has a sibling reference file under `skills/release-docs/` (e.g., `readme-update.md`, `release-update.md`) that specifies: which sections to update, what triggers an update (e.g., "new command added", "version change", "install behavior changed"), and model tier (Haiku for mechanical updates, Sonnet for complex rewrites). The `/document` command reads the registry, then loads only the reference files for artifacts that need updating.
- [ ] **AC4b (RELEASE.md update rules):** The `release-update.md` reference file instructs the agent to: add a row to the version table, update the "Current published release" line, update the roadmap table (mark shipped, project forward from backlog), and update the upgrade safety example. The agent reads the current `RELEASE.md` to match its format — does not invent a new structure.
- [ ] **AC4c (README.md update rules):** The `readme-update.md` reference file instructs the agent to: update the "What's new" section, update the "Current release line", add new flags/commands to setup examples, update the skills table and repo structure if changed, and extend the versioning section. Updates are scoped to sections affected by the feature — no full rewrites.
- [ ] **AC4d (Extensibility documentation):** The `## Artifact Registry` section in SKILL.md includes a brief guide on how to add a new documentation artifact: (1) create a sibling reference file with update rules, (2) add an entry to the registry table, (3) the `/document` command automatically picks it up. This serves as the template for other skills adopting the same pattern (e.g., ISS-034 backlog configurability).
- [ ] **AC5 (Skip when not needed):** If the feature does not affect user-facing behavior (e.g., internal refactors, test-only changes), the agent skips the artifact update and logs why. The trigger conditions in each reference file determine relevance.
- [ ] **AC6 (Verification):** Phase 7 verification includes a check that updated doc artifacts are consistent with the feature's ACs.
- [ ] **AC7 (No regression):** Existing Phase 7 behavior (CHANGELOG, release notes, CLAUDE.md) continues to work unchanged. The artifact registry is an addition, not a replacement.
- [ ] **AC8 (Version transparency):** All documentation artifacts updated by `/document` include a "Current as of: vX.Y.Z" line (or equivalent convention for that file type) so readers know which release the document reflects.

## Notes

### Current gap

The codingagents README documents repo structure, setup, commands, skills, and workflow. After landing ISS-009 (which wired 4 new commands through `resolve-feature.js`), the README was not updated to reflect this. The same will happen for every future feature unless `/document` is extended.

### Design considerations

- **Skill-level registry pattern:** The artifact registry lives in `skills/release-docs/SKILL.md`, loaded on demand by `/document`. This follows the project's progressive disclosure convention (code-review skill already uses sibling reference files for impact-analysis, reproduction, etc.). It avoids bloating CLAUDE.md's 250-line budget and keeps config colocated with the logic that uses it.
- **Extensibility template for other skills:** This pattern — registry table in SKILL.md + sibling reference files with per-entry rules — should serve as the template for ISS-034 (backlog configurability) and ISS-038 (multi-agent routing). Each skill owns its own config, loaded on demand.
- **Model escalation per artifact:** Each reference file specifies a model tier. Haiku for mechanical updates (CHANGELOG, release notes). Sonnet for complex rewrites (README sections, RELEASE roadmap). The `/document` command reads the model tier and escalates as needed.
- The agent should not re-read `src/` — it should derive changes from the PRD and the diff summary, keeping token cost low.
- `PIPELINE_GUIDE.md` and `QUICKSTART.md` can be added as registry entries when needed. The default registry starts with `README.md` and `RELEASE.md` only.
